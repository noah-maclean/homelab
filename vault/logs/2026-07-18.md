# Hermes

## Update Checker

Set up a daily cron job to check for Hermes Agent updates and deliver a summary to the Hermes Discord server (`#updates` channel).

**Setup:**
- **Schedule:** runs daily at **10:00 AM** (`0 10 * * *` crontab)
- **Mode:** `no_agent=True` — a standalone Python script runs each tick, zero LLM tokens consumed
- **Smart delivery:** if already on the latest version, the script outputs nothing → no message sent. If an update is available, a formatted summary gets delivered to the `#updates` channel.

**How it works:**
1. Fetches `hermes --version` to get current date tag (e.g. `v2026.7.1`)
2. Pulls latest releases from GitHub API (`NousResearch/hermes-agent` releases)
3. Compares date tags — if any newer releases exist, formats a summary
4. Uses emoji markers: 🐛 fixes, ⚠️ breaking changes, ✨ features, 🔒 security

**Script location:** `~/.hermes/scripts/hermes-update-check.py`
**Cron job ID:** `f4ae53358894`

**Current version at setup:** v0.18.0 (2026.7.1) — 2 newer releases available (v0.18.1, v0.18.2)

### Update — one-time notification

Updated the update checker script to only notify **once per release** instead of every day. Uses a state file at `~/.hermes/scripts/.hermes-update-state.json` to track the last announced tag. Future releases (e.g. v0.18.3) will trigger one notification and then stay silent until the next one comes out.
