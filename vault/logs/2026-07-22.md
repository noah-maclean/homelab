# 22/07/2026

## Mnemosyne Memory Provider Setup

Installed the [Mnemosyne](https://github.com/mnemosyne-oss/mnemosyne) external memory provider for Hermes Agent — a zero-dependency, SQLite-backed AI memory system with semantic search.

### Install Profile

- **`mnemosyne-hermes`** — Hermes plugin wrapper (v0.5.0)
- **`mnemosyne-memory[embeddings]`** — core library + fastembed for local vector generation (v3.14.0)
- Sits in the Hermes venv at `~/.hermes/hermes-agent/venv/`
- Local embeddings via ONNX Runtime (~800MB RAM, fine on 4GB LXC)

### Plugin Setup

1. `pip install "mnemosyne-hermes[embeddings]"` into the active Hermes venv
2. Symlinked `site-packages/mnemosyne_hermes/` → `~/.hermes/plugins/mnemosyne/`
3. Set `memory.provider: mnemosyne` in `~/.hermes/config.yaml`

### DB Location

`~/.hermes/mnemosyne/data/mnemosyne.db` — single SQLite file containing:

- Working memory (hot context, auto-injected before LLM calls)
- Episodic memory (long-term storage with hybrid search: 50% vector + 30% FTS5 + 20% importance)
- TripleStore (temporal knowledge graph with version chains)

### What Changes

- **Built-in memory (MEMORY.md/USER.md) still active** alongside — both work
- Mnemosyne automatically extracts facts, preferences, and patterns from conversation
- Semantic recall — can retrieve facts by meaning, not just keywords
- Importance scoring + temporal decay — critical info persists, trivial details fade
- Knowledge graph for structured relationships (e.g. "Noah → assigned_to → auth-migration")
- Multi-agent shared surface memory via `mnemosyne_shared_*` tools
- Cross-session sync available (optional, not enabled)

### Commands

```shell
source ~/.hermes/hermes-agent/venv/bin/activate
hermes memory status          # verify active provider
mnemosyne stats               # memory counts + DB path
mnemosyne sleep               # run consolidation
mnemosyne export backup.json  # export all memories
```

### Notes

- Works with any MCP client too (Cursor, Claude Code, Codex, Windsurf) via `mnemosyne mcp`
- Zero telemetry, zero cloud dependencies
- Takes effect on new Hermes sessions only (gateway restart or `/reset`)
- Client-side encryption available for sync (XChaCha20-Poly1305)
